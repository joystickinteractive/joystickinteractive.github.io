Best Practices

Clean Code

All code should look like one person typed it. Using the tools described above will help us achieve this goal.

The exact formatting of code isn’t nearly as important as consistency. To have readable code we rely on the machine. Do install the JsFormat package for Sublime Text and adjust the settings so that it automatically formats your JS anytime you save a file. To do this go to the Sublime Text 2 > Preferences > Package Settings > JsFormat > Settings - User menu and add {"format_on_save": true}. For the Atom editor the “format on save” preference is set to true by default.

There is also a way to configure your editor to enforce rules about line-endings (Win vs. Mac) as well as tab sizes and the like. We like such tools! Include an .editorconfig file in the root of your project. There are plugins for both Sublime Text and Atom.

Here are our configurations for hinting/linting with JsHint as well as our default editor config. Copy these over into existing projects and any new projects going forward.

Quality deliverables are essential for professionals. Sloppy or messy deliverables are unprofessional and reflect poorly on the final product and the team. Please remove legacy files, be certain the work is delivered in a clean file system, and in an orderly, logical structure that serves a clear purpose. Include a README file with every project.
